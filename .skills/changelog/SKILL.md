# Changelog skill
## Output format
## [version] — YYYY-MM-DD
### Added
- ...
### Changed
- ...
### Fixed
- ...
### Removed
- ...
## Rules
- Plain language. No jargon.
- One line per commit (group related commits).
- Skip merge commits and automated dependency bumps.
